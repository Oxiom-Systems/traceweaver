# Checklist

For each design decision:

- [ ] Decision ID is stable.
- [ ] Context is stated.
- [ ] Approved authority trace is present.
- [ ] Constraints are linked.
- [ ] Risks are linked.
- [ ] Alternatives are listed.
- [ ] Tradeoff rationale is stated.
- [ ] Consequences are stated.
- [ ] Affected requirements are listed.
- [ ] Derived requirements are captured as candidates.
- [ ] Interface/architecture impacts are listed.
- [ ] Verification and validation impacts are listed.
- [ ] Baseline impact is assessed.
- [ ] Decision status is explicit.

## Hold When

- [ ] Authority trace is missing.
- [ ] Decision changes requirement meaning.
- [ ] Alternatives are absent for a material decision.
- [ ] Risk or constraint impact is unknown.
- [ ] Downstream impacts are not assessed.
- [ ] Approval is inferred from implementation.
