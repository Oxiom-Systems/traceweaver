# Checklist

For each behavior:

- [ ] Behavior has stable ID or trace row.
- [ ] Parent need or source is known.
- [ ] Requirement or authority is approved.
- [ ] Requirement quality is accepted or passed.
- [ ] Design decision exists when implementation choice matters.
- [ ] Task links to approved authority.
- [ ] Implementation artifact is linked.
- [ ] Verification method is linked.
- [ ] Verification result or planned evidence is linked.
- [ ] Validation path is linked.
- [ ] Owner is identified.
- [ ] Status is current.
- [ ] Gaps, risks, and debt are visible.
- [ ] No dark-code candidate remains unexplained.

## Block When

- [ ] Behavior has no approved authority.
- [ ] Trace points only to a task.
- [ ] Trace points only to an idea or candidate need.
- [ ] Requirement failed quality review without accepted risk/gap.
- [ ] Verification is missing for implemented behavior.
- [ ] User-facing behavior has no validation path or approved deferred plan.
