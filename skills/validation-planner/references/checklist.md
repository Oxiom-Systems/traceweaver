# Checklist

For each need or release objective:

- [ ] Need ID or source intent is linked.
- [ ] Intended use is described.
- [ ] Operating context is described.
- [ ] Stakeholder, rater, representative user, or approved proxy is named.
- [ ] Rater role is recorded.
- [ ] Rater relationship to the work is recorded.
- [ ] Scenario reflects the intended context.
- [ ] Acceptance signal is observable.
- [ ] Evidence collection method is defined.
- [ ] Verification evidence is separate and linked where relevant.
- [ ] Adoption evidence or approval package inputs are defined.
- [ ] Deferred validation has owner, trigger, expected evidence, and approval.

## Hold When

- [ ] Implementer self-assessment is the only evidence.
- [ ] Rater is unnamed.
- [ ] Scenario is not representative.
- [ ] Acceptance criteria are vague.
- [ ] Evidence path is missing.
- [ ] Deferred validation has no approved gap or trigger.
