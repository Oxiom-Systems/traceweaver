# Checklist

For each stakeholder input:

- [ ] Original wording preserved.
- [ ] Stakeholder or source identified.
- [ ] Intended user/operator identified.
- [ ] Context or operating situation captured.
- [ ] Problem/opportunity captured.
- [ ] Desired outcome captured.
- [ ] Success signal captured.
- [ ] Failure signal captured when available.
- [ ] Assumptions separated.
- [ ] Constraints separated.
- [ ] Risks separated.
- [ ] Design suggestions separated.
- [ ] Candidate needs have IDs.
- [ ] Candidate requirements have IDs.
- [ ] Candidate/draft status is explicit.
- [ ] Open questions are listed.
- [ ] No item is marked approved by inference.

## Stop And Ask When

- [ ] Stakeholder is unknown.
- [ ] Desired outcome is unclear.
- [ ] Original wording conflicts with reframed text.
- [ ] A design suggestion may be a hard constraint.
- [ ] Success cannot be recognized.
- [ ] The agent would need to invent scope.
