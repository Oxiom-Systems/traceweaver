# Checklist

For each route decision:

- [ ] Current artifact type is identified.
- [ ] Requested action is identified.
- [ ] Current lifecycle state is identified.
- [ ] Authority state is known.
- [ ] Baseline reference is known when relevant.
- [ ] Traceability state is known when relevant.
- [ ] Verification state is known when relevant.
- [ ] Validation state is known when relevant.
- [ ] Risk/gap/change state is known when relevant.
- [ ] Blocking conditions are listed.
- [ ] Next skill is the narrowest owner of the next question.
- [ ] Required inputs for next skill are listed.
- [ ] Handoff record preserves source and context.
- [ ] The route does not bypass a hold.

## Hold When

- [ ] Authority state is unknown and implementation is requested.
- [ ] Required source context is missing.
- [ ] Weak requirement has not been reviewed.
- [ ] Traceability gap is uncontrolled.
- [ ] V&V evidence is required but unplanned.
- [ ] Baseline or gate state is stale.
- [ ] No skill has enough inputs to proceed.
