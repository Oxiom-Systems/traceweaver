# Source Basis

This public plugin package intentionally does not publish licensed, controlled,
internal, or purchase-only source names. Runtime agents must use the distilled
TraceWeaver rules in this skill, not source text or source-specific summaries.

## Public Package Boundary

The requirements-reviewer rules are original TraceWeaver guidance informed by
general systems-engineering practice and public training material. Protected
source names, page references, extraction notes, and internal provenance are
kept out of the public plugin package.

Public examples may be cited only when the user provides them or asks for source
context. Even then, the agent must use original TraceWeaver wording and avoid
copying source tables, diagrams, examples, or long passages.

## Runtime Boundary

- Use source context only to justify the existence of the distilled model.
- Do not quote protected source text in runtime findings.
- Do not copy source tables, source diagrams, page layouts, or examples.
- Mermaid diagrams in runtime material must be new TraceWeaver diagrams.
- Public citations may point to official public pages, but the rule text must
  remain original.

## Update Rule

When adding a new source to a non-public provenance record, record:

- source ID
- exact source name
- access type
- what operational rule family it informed
- whether it is public, licensed/internal, or controlled/purchase-only
- any reuse restriction relevant to runtime agent material

Do not add protected source names or internal provenance details to this public
runtime package without a new public-hygiene review.
